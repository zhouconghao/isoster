## Problems / Blockers

  photutils v2.x — fixed in masking.py.
- `photutils` is only in `dev` extras; example requires `uv sync --extra dev`.
  README documents this implicitly but no explicit warning in run_example.py.
- Data files (`eso243-49.fits`, `ngc3610.fits`) are untracked in the main repo —
  they are in `examples/data/` and git-ignored via `*.fits`? Check `.gitignore`
  for explicit exclusion policy. For the worktree they were symlinked from main.

## Key Decisions
- Used `on_galaxy_box=32, npixels=10, dilate_fwhm=8.0` for both galaxies instead of
  plan-specified `box=4` — the small box caused ~80% masking by following galaxy disk
  gradients too tightly.
- `plot_qa_summary_extended()` is a new public function (not replacing existing
  `plot_qa_summary`) for backward compatibility.
- Baseline conditions still use `harmonic_orders=[3,4]` in config for post-hoc
  computation; `simultaneous_harmonics=False` means they don't feed back into
  geometry fitting.
