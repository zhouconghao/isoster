### Phase 2: Modify fit_isophote() loop (DONE)
Core changes in `fit_isophote()` in `isoster/fitting.py`:
1. **Pre-loop setup**: `isofit_min_points` (1 + 2*(2+len(orders))), `best_isofit_harmonics_stored` flag
2. **Per-iteration fallback**: `use_isofit_this_iter` decided after sigma clipping. Falls back to 5-param when `6 <= points < isofit_min_points`, with RuntimeWarning emitted once per isophote.
3. **Dual-path harmonic fitting**: `fit_all_harmonics()` vs `fit_first_and_second_harmonics()`
4. **Dual-path model eval**: `evaluate_harmonic_model()` vs `harmonic_function()` — ISOFIT RMS excludes higher-order signal
5. **5x5 covariance sub-matrix**: `cov_matrix[:5, :5]` passed to `compute_parameter_errors` in ISOFIT mode
6. **Harmonics stored during iteration**: When `use_isofit_this_iter` and best_geometry updated, coefficients extracted and normalized by `sma * |gradient|`, errors from covariance diagonal
7. **Post-convergence guards**: All three blocks (convergence, geometry-stability, max-iter) guarded by `best_isofit_harmonics_stored`

## Key Decisions
1. **No numba for ISOFIT path**: Variable-width matrices negate JIT benefits; numpy lstsq is fast enough
2. **Per-iteration fallback**: `use_isofit_this_iter` decided after sigma clipping (not pre-loop), so heavy masking mid-galaxy triggers fallback gracefully
3. **Harmonics stored during iteration**: When best_geometry is updated and ISOFIT is active, coefficients are stored immediately (not just at convergence), so the best harmonics are always from the best geometry
4. **Post-hoc fallback**: If ISOFIT couldn't activate (small SMA), the existing `fit_higher_harmonics_simultaneous()` post-hoc path is used as fallback
5. **Geometry updates unchanged**: `A1, B1, A2, B2 = coeffs[1:5]` — same indexing in both paths, no geometry logic changes
