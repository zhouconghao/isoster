    4. Decisions

    • Decision: enforce method-specific 2-D model reconstruction in QA pipelines.
    • Rationale: model builder should match extraction method; previously photutils profiles were reconstructed with isoster model code.
    • Decision: harmonize method_cog_flux with fallback precedence to avoid QA point loss when tflux_e is sparse.
    • Rationale: CoG in QA should have one-to-one valid correspondence with finite 1-D profile rows when CoG information exists.
